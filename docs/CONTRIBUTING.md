# المساهمة والدعم

هذا المشروع مفتوح المصدر، وأي مساهمة تساعد على تحسين الجودة والسهولة وسرعة التطوير.

## كيف تدعم المشروع؟

1. ارفع مشكلة (`Issue`) عند وجود خلل أو اقتراح ميزة.
2. أضف اقتراحات واضحة: ما الذي تتوقعه؟ وما الذي يحدث فعليًا؟
3. راجع طلبات الدمج (`Pull Request`) وشارك بملاحظاتك.
4. إذا أعجبك المشروع، شاركه/ضع له نجمة لمساعدتنا في الوصول.

## كيف أساهم كمطوّر؟

### 1) تشغيل المشروع محليًا

شغّل المشروع ثم افتح الواجهة:

```bash
npm install
npm run dev
```

إذا كنت ستستخدم توليد البرومبتات عبر Gemini، أضف ملف `.env.local`:

```bash
VITE_GEMINI_API_KEY=your_gemini_api_key_here
```

ملاحظة: في الإنتاج يتم استدعاء Gemini عبر المسار الخلفي `/api/vibe/generate` لتجنب كشف مفتاح الـ API في المتصفح.

### 2) قبل إنشاء أي تغيير

شغّل:

```bash
npm run lint
```

وحاول الالتزام بعقد JSON كما هو موثق في `docs/GEMINI_PROMPT_CONTRACT.md`.

### 3) إضافة/تعديل وضعيات (Modes) أو القوالب

مصدر إعدادات الوضعيات هو:

- `src/config/vibeModeConfig.json`

عند إضافة Quick Templates أو تعديل `goals` و `focus`:

1. حافظ على مفاتيح JSON كما هي (لا تغيّر أسماء الحقول الأساسية بدون تحديث التوثيق).
2. تأكد أن `promptType` المدعوم موجود ضمن نفس المفاتيح.
3. لا تغيّر صيغة العقد الناتج دون تحديث `docs/GEMINI_PROMPT_CONTRACT.md`.

### 4) تعديل وثائق المنصة

صفحات الوثائق تُعرض داخل واجهة الموقع من ملفات `docs/*.md`.

لإضافة صفحة جديدة:

1. أنشئ ملف Markdown داخل `docs/`
2. سجّله داخل `src/App.jsx` ضمن `docsItems`
3. راجع اتجاه النص: العربية RTL، بينما كتل الكود تعرض LTR تلقائيًا.

## إرشادات RTL/LTR داخل الوثائق

- النص العربي يجب أن يبقى RTL.
- كتل الكود (`inline code` و `fenced code`) يجب أن تبقى LTR لتسهيل قراءة الأوامر والمسارات.
